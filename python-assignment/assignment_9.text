Answers

1. To what does a relative path refer?

    Ans-1: A relative path refers to a location relative to the current working directory.

2. What does an absolute path start with in your operating system?

    Ans-2: An absolute path starts from the root directory (/ on Unix/Linux/Mac or a drive letter like C:\ on Windows).

3. What do the functions os.getcwd() and os.chdir() do?

    Ans-3: os.getcwd() returns the current working directory, and os.chdir() changes the current working directory.

4. What are the . and .. folders?

    Ans-4: . refers to the current directory, and .. refers to the parent directory.

5. In C:\bacon\eggs\spam.txt, which part is the dir name, and which part is the base name?

    Ans-5: C:\bacon\eggs\ is the directory name, and spam.txt is the base name.

6. What are the three “mode” arguments that can be passed to the open() function?

    Ans-6: The three modes are:
        'r' for reading,
        'w' for writing,
        'a' for appending.

7. What happens if an existing file is opened in write mode?

    Ans-7: Opening an existing file in write mode ('w') will erase the file’s contents.

8. How do you tell the difference between read() and readlines()?

    Ans-8: read() returns the entire file content as a single string, while readlines() returns a list of lines from the file.

9. What data structure does a shelf value resemble?

    Ans-9: A shelf value resembles a dictionary, where keys and values can be stored persistently.
